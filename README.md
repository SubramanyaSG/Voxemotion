## Project Overview <a name="overview"></a>

VoxEmotion is an end-to-end **Text-to-Speech system with emotion control** built using:

| Component | Technology |
|---|---|
| Speech synthesis | NVIDIA Tacotron2 (torch.hub) + WaveGlow |
| Emotion classifier | CNN-LSTM trained on ESD mel-spectrograms |
| Dataset | Emotional Speech Dataset (ESD) – 10 speakers × 5 emotions |
| Web UI | Flask + HTML5 Audio API + Canvas waveform |
| File parsing | PyPDF2, python-docx, pdfminer |

### Option B – Manual (Step-by-Step)

#### Step 1 – Install Python 3.10.11

Download from: https://www.python.org/downloads/release/python-31011/

During installation:
- ✅ Check **"Add Python to PATH"**
- ✅ Check **"Install for all users"**

Verify:
```cmd

```

#### Step 2 – Open Command Prompt in project folder

```cmd
cd "D:\7th Sem\Major Project - 2\Code\ets\tts_emotion_project"
```

#### Step 3 – Create and activate virtual environment

```cmd

```

You should see `(venv)` in your prompt.

#### Step 4 – Upgrade pip

```cmd

```

#### Step 5 – Install PyTorch (CPU version)

```cmd

```

> For NVIDIA GPU (CUDA 11.8):
> ```cmd
> pip install torch==2.1.0 torchaudio==2.1.0 --index-url https://download.pytorch.org/whl/cu118
> ```

#### Step 6 – Install remaining dependencies

```cmd
pip install -r requirements.txt
```

#### Step 7 – Register Jupyter kernel

```cmd
python -m ipykernel install --user --name=voxemotion --display-name "VoxEmotion (Python 3.10)"
```

#### Step 8 – Launch Jupyter Notebook

```cmd
jupyter notebook
```

This opens your browser at `http://localhost:8888`.

## Dataset Setup <a name="dataset-setup"></a>

Your dataset should be organized as:

```
D:\7th Sem\Major Project - 2\Code\ets\Emotion Speech Dataset1\English\
├── 0011\
│   ├── angry\
│   │   ├── 0011_000351.wav
│   │   └── ...
│   ├── happy\
│   │   └── ...
│   ├── neutral\
│   ├── sad\
│   ├── surprise\
│   └── 0011.txt          ← tab-separated: filename \t text \t emotion
├── 0012\
│   └── ...
...
└── 0020\
```

**The `.txt` file format** (tab-separated):
```
0011_000351	I am so angry right now	Angry
0011_000352	This is frustrating	Angry
```

> If your `.txt` files use a different format (space-separated, etc.), the parser will still attempt extraction. The emotion will fall back to the folder name.

### In Jupyter Notebook

1. Navigate to `notebooks/TTS_Emotion_Control.ipynb`
2. **Select kernel:** Kernel → Change Kernel → **VoxEmotion (Python 3.10)**

| Cell # | What it does |
|---|---|

## What Each Notebook Cell Does <a name="notebook-cells"></a>

### Cell 3 – Dataset Scan
- Walks all speaker folders (0011–0020)
- Parses `.txt` files for transcripts
- Tests each `.wav` file with `soundfile.info()`
- Prints readable/unreadable count, total duration, per-speaker breakdown

### Cell 4 – Trial Readability
- Loads 2 audio files per emotion with `librosa`
- Verifies sample rate, number of samples
- Displays inline audio player in Jupyter

### Cell 5 – Mel-Spectrogram Visualization
- Computes mel-spectrogram using `librosa.feature.melspectrogram`
- Plots in dB scale with `librosa.display.specshow`
- Shows waveform beside each mel-spec
- Saves PNG to `outputs/mel_spectrograms.png`

### Cell 6 – Feature Extraction
- Extracts 80-band mel features, padded/truncated to 300 frames
- Returns `X` array of shape `(N, 80, 300)`

### Cells 7–9 – Training
- CNN extracts spatial features from mel-spec image
- LSTM captures temporal emotion patterns
- Best checkpoint saved to `models/emotion_clf_best.pth`
- Classification report shows per-emotion precision/recall/F1

### Cell 10 – Tacotron2 Setup
- Attempts NVIDIA Tacotron2 + WaveGlow download (requires internet)
- Falls back gracefully to retrieval-based synthesis if download fails
- Emotion prosody applied: pitch shift, energy scale, time stretch

## Troubleshooting <a name="troubleshooting"></a>

### `numba` installation fails
```cmd
pip install numba==0.58.1 --no-build-isolation
```

### `librosa` import error
Ensure `numba` and `llvmlite` are compatible:
```cmd
pip install llvmlite==0.41.1 numba==0.58.1
```

### Tacotron2 download fails (no internet / firewall)
The system automatically falls back to retrieval-based synthesis using actual ESD audio files with emotion transforms applied. This works completely offline.

### Port 5000 already in use
Edit the last line of `app.py`:
```python
app.run(host='0.0.0.0', port=5001, ...)
```

### `torch.hub` SSL error
```python
import ssl
ssl._create_default_https_context = ssl._create_unverified_context
```
Add this before the torch.hub.load() call in the notebook.

### Jupyter kernel not showing "VoxEmotion"
```cmd
venv\Scripts\activate
python -m ipykernel install --user --name=voxemotion --display-name "VoxEmotion (Python 3.10)"
```

### `soundfile` cannot read .wav
```cmd
pip install soundfile==0.12.1 --force-reinstall
```
